# ft_printf 概要

`ft_printf` は `printf` の主要な変換指定子を再現した静的ライブラリです。可変長引数（引数の数が決まっていない関数）を使用してフォーマット文字列を解析し、各指定子に応じた出力を行います。

本物の `printf` と同様に、画面に文字を出力するだけでなく、**「実際に表示された合計文字数」**を戻り値として正確に返す仕様になっています。

## 可変長引数の仕組みと `va_macro` の解説

`ft_printf` では、何個渡されるか分からない引数を順番に読み出すために、`<stdarg.h>` に定義されている `va_list` と3つの重要なマクロを使用しています。これらはメモリ上に並んだ引数を指し示す「矢印（ポインタ）」のように動作します。



### 1. `va_list`
可変長引数のリスト（メモリ上の位置）を保持するための専用の型です。引数を順番に指し示す「矢印」の役割を果たします。

### 2. `va_start(ap, format)`
可変長引数の処理を開始するためのマクロです。
矢印（`ap`）を、固定引数である `format` のすぐ後ろ（最初の可変長引数の位置）にセットして初期化します。

### 3. `va_arg(ap, 型)`
矢印（`ap`）が現在指している場所から、指定した「型」のサイズ分のデータを取り出します。
データを取り出した後、矢印（`ap`）は**自動的に次の引数の位置へと進みます**。このマクロを繰り返すことで、すべての引数を順番に処理できます。

### 4. `va_end(ap)`
可変長引数の処理を安全に終了するためのマクロです。
使い終わった矢印（`ap`）を無効化し、メモリのクリーンアップ（後片付け）を行います。

### 対応している変換指定子

| 指定子 | 出力内容 |
|--------|----------|
| `%c` | 1文字 |
| `%s` | 文字列（NULL の場合は `(null)` を表示） |
| `%d` / `%i` | 符号付き10進整数 |
| `%u` | 符号なし10進整数 |
| `%x` | 符号なし16進数（小文字） |
| `%X` | 符号なし16進数（大文字） |
| `%p` | ポインタアドレス（`0x` プレフィックス付き、NULL の場合は (nil) を表示） |
| `%%` | `%` 文字そのもの |

戻り値は出力した文字数。

`write()` が失敗した場合は `-1` を返します。

### アルゴリズムとデータ構造

フォーマット文字列を1文字ずつ走査し、`%` を検出したら次の文字を `handle_format` 関数に渡して振り分けます。全ヘルパー関数に `int *count` ポインタを渡すことで、文字数の追跡とエラーの伝播を一元管理しています。数値出力には再帰関数を使用し、一時バッファなしで桁の取り出しを行っています。

## （例）main

#include "ft_printf.h"
#include <stdio.h>

int main(void)
{
    // 文字
    ft_printf("char: %c\n", 'A');

    // 文字列
    ft_printf("string: %s\n", "hello");

    // 数字
    ft_printf("int: %d\n", 42);
    ft_printf("int: %i\n", -42);

    // 符号なし
    ft_printf("unsigned: %u\n", 100);

    // 16進数
    ft_printf("hex lower: %x\n", 255);
    ft_printf("hex upper: %X\n", 255);

    // ポインタ
    int n = 42;
    ft_printf("pointer: %p\n", &n);

    // パーセント
    ft_printf("percent: 100%%\n");

    // --------------- NULLのテスト ------------------

	ft_printf("自作 %%s NULL: %s\n", (char *)NULL);
	printf("本物 %%s NULL: %s\n", (char *)NULL);

	ft_printf("自作 %%p NULL: %p\n", NULL);
	printf("本物%%p NULL: %p\n", NULL);

	return (0);
}
