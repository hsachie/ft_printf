# ft_printf

## 概要 (Description)
C言語の標準ライブラリ関数である `printf` を再実装したカスタム関数 `ft_printf` を作成するプロジェクトです。
可変長引数（Variadic arguments）の扱いを深く理解し、書式指定子を適切に解析してフォーマット出力を行うクリーンなコード設計を目的としています。

### 対応している書式指定子
本プロジェクトでは、以下の基本変換（Mandatory部分）を完全にサポートしています。

| 指定子 | 説明 |
|:---:|---|
| `%c` | 単一文字を出力 |
| `%s` | 文字列を出力 |
| `%p` | void型ポインタのアドレスを十六進数表記で出力 |
| `%d` | 符号付き十進整数を出力 |
| `%i` | 符号付き十進整数を出力 |
| `%u` | 符号無し十進整数を出力 |
| `%x` | 符号無し十六進整数（小文字）を出力 |
| `%X` | 符号無し十六進整数（大文字）を出力 |
| `%%` | パーセント記号そのものを出力 |

アルゴリズムとデータ構造の解説 (Algorithm & Data Structures)
1. 書式指定子の解析アルゴリズム（スキャンと条件分岐）
ft_printf は、渡された文字列（フォーマット文字列）を1文字ずつ走査し、% を見つけたときに状態を切り替えるループ処理で構成されています。

テキスト走査: 文字列を先頭からループで走査し、% 以外の文字はそのまま write 関数で出力し、出力文字数を加算します。

ディスパッチ処理: % に遭遇した場合、次の文字（または連続するフラグ文字）を読み込み、どの書式指定子（c, s, p など）に該当するかを条件分岐で判別します。

引数の取り出し: 判別した指定子に応じて専用の出力関数を呼び出し、可変長引数から対応するデータを取得してフォーマット出力を行います。

2. データ構造と技術選定
本プロジェクトでは、シンプルさとメモリ効率を最優先し、以下の技術的選定を行いました。

可変長引数マクロ（stdarg.h）: 関数の引数の数が動的に変化するため、C言語標準の va_list 構造体を利用しました。これにより、スタック上の引数領域に安全にアクセスし、型に応じたデータ抽出を行っています。

構造体によるフラグ管理（※ボーナス対応時）: 各種フラグ（幅、精度、左寄せ、ゼロ埋めなど）の状態を一時的に保持するため、専用の t_flags 構造体を定義しました。条件分岐をバラバラの変数で管理するよりも、構造体にカプセル化して各関数にポインタで引き回す方が、コードの可読性と拡張性が圧倒的に高くなると判断しました。

出力文字数の同期（戻り値の累積）: printf は「出力した総文字数」を返却する必要があります。グローバル変数の使用を避けるため、すべての下位出力関数が「自身が出力したバイト数」を返却し、それを親関数（ft_printf）側で累積加算していく設計を採用しました。これにより、安全かつ正確な文字数カウントを実現しています。
